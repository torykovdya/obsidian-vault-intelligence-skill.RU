## Откуда берётся Zettelkasten логика

Скилл реализует три ключевых принципа Луманна:

### Принцип 1: Атомарность
Луман хранил ровно одну идею на карточке. Скилл проверяет каждую заметку по 5 критериям и переписывает её, если она не атомарна.

**Почему это важно:** атомарные заметки можно переиспользовать в разных контекстах. Блок заметок — нет.

### Принцип 2: Контекстные теги
Луман не спрашивал "куда это положить?", он спрашивал "когда я захочу это найти?". Скилл генерирует теги второго типа.

**Пример:**
- Плохой тег: `#ai` (категория хранения)
- Хороший тег: `#when-building-rag-pipelines` (контекст поиска)

### Принцип 3: Компаундирование через связи
Ценность vault растёт не линейно с количеством заметок, а экспоненциально с количеством связей. Скилл предлагает связи при каждом анализе.

**Математика Луманна:** 90,000 заметок × средние связи = база для 58 книг и 600 статей.

## Логика Vault Intelligence (Level 2)

### Phase 1: Тематическая экстракция

Скилл ищет:
- **Рекуррентные концепты** — слова/идеи, которые появляются в 3+ заметках
- **Кластеры** — группы концептов, которые часто встречаются вместе
- **Тренды** — темы, плотность которых растёт
- **Одиночки** — важные идеи, упомянутые раз и забытые

Это работает даже без явных ссылок `[[link]]` — только через семантический анализ текста.

### Phase 2: Карта компетенций

Скилл использует таксономию из `references/ai-creator-taxonomy.md` — список из ~20 ключевых навыков для AI-создателей — и сопоставляет с кластерами vault.

**Логика оценки глубины:**
- 1 заметка по теме → упоминание
- 3–5 заметок → знакомство
- 5–10 заметок → рабочее знание
- 10+ заметок с перекрёстными ссылками → экспертиза

### Phase 3: Opportunity Intelligence

Использует 15 паттернов из `references/opportunity-patterns.md`. Каждый паттерн — это сигнал в vault, который указывает на конкретный тип возможности.

**Пример паттерна:** "Dense Cluster" — 5+ заметок на одну тему → предложение написать definitive guide именно по этой теме.

### Phase 4: Прогноз траектории

Логика прогноза:
1. Найти сильнейший кластер → это текущая экспертиза
2. Найти самый быстрорастущий кластер → это направление развития
3. Найти критический пробел → это то, что нужно изучить
4. Совместить (1) + (3) → это то, чем пользователь станет через 6 месяцев
5. Сопоставить с архетипами из таксономии → конкретный "кем ты станешь"

### Phase 5: Health Check

Проверяет структурное здоровье vault по 6 метрикам:
- % атомарных заметок
- Количество orphan notes (без ссылок)
- Наличие MOC-заметок
- Pipeline Fleeting → Permanent
- Качество тегов (контекстные vs категориальные)
- Последняя дата активности

## Почему именно для AI-энтузиастов

Скилл добавляет "AI Creator Lens" — три дополнительных теста для каждого кластера:

**Build It Test:** может ли этот кластер знаний стать инструментом или агентом?
→ Ищет структуры "input → process → output" в заметках

**Teach It Test:** достаточно ли знаний, чтобы учить других?
→ Оценивает глубину и уникальность угла

**Compound It Test:** какие кластеры создают редкую комбинацию?
→ Ищет пересечения, которых нет у большинства людей в нише

## Ограничения

**Что скилл НЕ делает:**
- Не читает vault напрямую (нет интеграции с Obsidian API — пока)
- Не хранит историю ваших vault'ов между сессиями
- Не заменяет вашу собственную работу по созданию связей

**Что влияет на качество анализа:**
- Больше заметок = точнее паттерны
- Даты на заметках = точнее тренды
- Ваши слова (не цитаты) = точнее понимание вашего мышления
