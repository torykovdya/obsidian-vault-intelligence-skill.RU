# Как использовать Obsidian Vault Intelligence эффективно

> Инструмент настолько хорош, насколько хорош ваш ввод. Это руководство покажет, как получить максимум.

---

## Принцип № 1: Мусор на входе — мусор на выходе

Claude анализирует то, что вы даёте. Чем лучше структурированы ваши исходные заметки, тем точнее анализ. Но даже плохие заметки — это материал для работы. Не ждите идеального vault.

---

## Сценарии использования

### 🔸 Сценарий А: Разбор одной заметки

**Когда использовать:** только что написали заметку и хотите её улучшить

**Как подать:**
```
У меня есть заметка из Obsidian. Разбери её:
- Какой это тип заметки?
- Насколько она "атомарна"?
- Перепиши её как Permanent Note
- Предложи связи с другими концептами
- Какие теги поставить?

--- ЗАМЕТКА ---
[текст заметки]
```

**Что получите:** переработанная заметка, 3–5 связей, умные теги, место в vault

---

### 🔸 Сценарий Б: Переработка пачки заметок

**Когда использовать:** накопилось 10–50 необработанных заметок в Inbox

**Как подать:**
```
Вот мои необработанные заметки из папки Inbox.
Для каждой:
1. Определи тип (Fleeting / Literature / Permanent)
2. Оцени качество по 5-балльной шкале
3. Перепиши в Permanent Note формат те, что стоят этого
4. Остальные — скажи, что с ними делать

--- ЗАМЕТКИ ---
[заметка 1]
---
[заметка 2]
---
[заметка 3]
```

**Лайфхак:** добавьте дату каждой заметки — Claude лучше определит тренды

---

### 🔸 Сценарий В: Анализ тематического кластера

**Когда использовать:** хотите понять, насколько глубоко разбираетесь в теме

**Как подать:**
```
Вот все мои заметки по теме [AI-агенты / промптинг / автоматизация].
Построй карту этой темы:
- Что я знаю хорошо?
- Где пробелы?
- Какой контент или продукт я мог бы создать на основе этого?

--- ЗАМЕТКИ ПО ТЕМЕ ---
[все заметки по теме]
```

---

### 🔸 Сценарий Г: Полный Vault Intelligence

**Когда использовать:** раз в месяц для стратегического обзора

**Как экспортировать vault:**

*Метод 1 — вручную (для небольших vault, до 30 заметок):*
1. Откройте папку vault в файловом менеджере (Finder / Explorer)
2. Выберите нужные `.md` файлы — например, всё из папки `10-Permanent/`
3. Откройте каждый файл и скопируйте содержимое в один текстовый документ
4. Вставьте в Claude

> ⚠️ Функции "Export to PDF" для всего vault в Obsidian нет. Не ищите её — используйте один из методов ниже.

*Метод 2 — через терминал:*
```bash
# Собрать все .md файлы vault в один файл
find ~/ObsidianVault -name "*.md" -exec cat {} \; > vault-export.txt
```

*Метод 3 — Python-скрипт (в папке /examples):*
```bash
python3 examples/export-vault.py ~/ObsidianVault > vault-export.txt
```

**Как подать:**
```
Вот экспорт моего Obsidian vault за последние 3 месяца.
Сделай полный Vault Intelligence анализ:
1. Карту знаний (кластеры, слепые зоны)
2. Карту компетенций
3. Топ-3 статьи, которые я могу написать
4. Топ-3 продукта / автоматизации
5. Прогноз на 6 месяцев

[содержимое vault]
```

## Как подготовить заметки для лучшего анализа

### Минимальная подготовка (2 минуты)
Добавьте в начало каждой заметки:
```
Date: 2024-01-15
Topic: AI Agents
Type: fleeting / literature / permanent
```

### Оптимальная подготовка (5 минут на пачку)
Структура заметки, которую Claude анализирует лучше всего:
```markdown
# Заголовок (одна мысль)

## Идея
[суть в 1-3 предложениях своими словами]

## Источник
[откуда идея: книга / видео / личный опыт]

## Связи
- похоже на [[другая идея]]
- противоречит [[третья идея]]

## Применение
[как это можно использовать]

Tags: #промптинг #когда-строю-агентов
Date: 2024-01-15
```

## Промпты для продвинутых задач

### Найти, о чём писать прямо сейчас
```
Вот мои заметки. Мне нужна идея для статьи/поста,
которую я могу написать на этой неделе.
Ищи: плотный кластер заметок + мой уникальный угол + востребованная тема.
Дай топ-3 идеи с объяснением, почему именно я должен это написать.

[заметки]
```

### Обнаружить слепые зоны
```
Я занимаюсь [AI автоматизацией / контентом про LLM / etc.].
Вот мои заметки. Что важного я НЕ изучаю?
Что должно быть в моём vault, но отсутствует?
Что я упускаю, что мои конкуренты, вероятно, знают?

[заметки]
```

### Спланировать обучение на месяц
```
Вот мои текущие заметки и уровень знаний.
Составь план обучения на следующие 4 недели:
- Что изучать (конкретно, не абстрактно)
- В каком порядке (что разблокирует что)
- Как это пополнит мой vault
- Какие возможности откроются после

[заметки + текущий контекст]
```

### Превратить vault в продукт
```
Я хочу создать [курс / инструмент / сервис / newsletter].
Вот мои заметки. Что из накопленного знания уже можно упаковать?
- Какой минимально жизнеспособный продукт я могу сделать за 2 недели?
- На основе каких именно заметок?
- Кому это нужно?

[заметки]
```

## Частые ошибки

### ❌ "Мои заметки не готовы / они в хаосе"
**Реальность:** хаотичные заметки — это нормальный ввод. Скилл создан именно для этого. Отправляйте как есть.

### ❌ Отправлять по одной заметке бесконечно
**Лучше:** собрать 20–30 заметок и отправить вместе. Паттерны видны только в массиве.

### ❌ Игнорировать предложенные связи
**Важно:** когда Claude предлагает `[[связь с концептом X]]` — создайте эту заметку в Obsidian. Именно так растёт граф.

### ❌ Не возвращаться к анализу
**Лучше:** делайте Vault Intelligence раз в месяц. Сравнивайте карты компетенций. Смотрите, как вы растёте.

### ❌ Копировать чужие заметки в vault
**Zettelkasten-принцип:** только свои слова. Копирование = иллюзия знания, не знание.

## Рабочий ритм (рекомендованный)

```
ЕЖЕДНЕВНО (5–10 мин)
└── Обработать Inbox: fleeting notes → permanent notes
    Промпт: "Переработай эти заметки в Permanent Notes"

ЕЖЕНЕДЕЛЬНО (15–20 мин)
└── Анализ кластера недели
    Промпт: "Что я узнал на этой неделе? Какие связи?"

ЕЖЕМЕСЯЧНО (30–45 мин)
└── Полный Vault Intelligence
    Промпт: полный анализ vault, обновление карты компетенций

КВАРТАЛЬНО (1–2 часа)
└── Стратегический обзор
    Промпт: "Что изменилось за 3 месяца? Что создавать дальше?"
```

## Интеграция с Obsidian

### Шаблон для быстрого экспорта
Создайте заметку `Export-Template.md` в vault:
```markdown
---
export: true
date: {{date}}
cluster: {{cluster}}
---
```

### Плагины, которые помогают
- **Dataview** — SQL-запросы к vault, помогает собрать заметки по теме
- **Templater** — автоматизация шаблонов заметок
- **Graph Analysis** — визуализация связей
- **Smart Connections** — AI-поиск похожих заметок

### Команда для сбора заметок по тегу (Dataview)
```dataview
LIST
FROM #ai-agents
SORT file.mtime DESC
```

## Метрики прогресса

Отслеживайте раз в месяц:

| Метрика | Как считать |
|---------|------------|
| Permanent Notes | Количество файлов в `/10-Permanent/` |
| Средние связи на заметку | Считает Claude при анализе |
| Активных кластеров | Темы с 5+ заметками |
| Orphan notes | Заметки без единой ссылки |
| Новые заметки/неделю | Лог в `Intelligence/Trajectory.md` |

> Луман создавал 6 permanent notes в день. Ваша цель — не 6, а последовательность. Даже 3 заметки в неделю за год — это 150 постоянных идей.

## Следующий шаг

1. Установите скилл (файл в `/releases`)
2. Откройте Obsidian, скопируйте 10 последних заметок
3. Отправьте в Claude с промптом из Сценария Б
4. Создайте в vault файл `Intelligence/Competency-Map.md` на основе результата
5. Повторите через месяц и сравните

Ваш vault — это ваш интеллектуальный капитал. Скилл просто делает его видимым.
